# SMS segment integration brief

## Required product changes

1. replace message-count language with segment-count language
2. default included allotment: `1,000 segments included`
3. show live segment meter in SMS composer
4. show remaining segments in billing/usage surfaces
5. show soft warning at ~80% usage

## Critical UX copy

- `142 / 160 characters`
- `1 segment`
- `987 segments remaining`
- `You're running low on text credits`

## Why

If the UI hides segmentation, users will accidentally send expensive messages and blame the product later.
